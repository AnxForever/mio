# Mio ADR Draft Pack

This directory contains draft Architecture Decision Records extracted from the architecture research pass in `docs/research/architecture-review-long-task.md`.

Status convention:

- `Proposed`: recommended by research, not yet formally accepted.
- `Accepted`: implemented and ratified by maintainers.
- `Superseded`: replaced by a later ADR.

Current drafts:

- [ADR-0001: Local-First Modular Monolith](./0001-local-first-modular-monolith.md)
- [ADR-0002: Localhost-First Security Model](./0002-localhost-first-security-model.md)
- [ADR-0003: Persona Source Hierarchy](./0003-persona-source-hierarchy.md)
- [ADR-0004: Emotion State Ownership](./0004-emotion-state-ownership.md)
- [ADR-0005: Package Drift Policy](./0005-package-drift-policy.md)

Use these as decision baselines before code refactors. They are intentionally conservative: Mio is currently rated `good, not excellent`, so these ADRs document trade-offs instead of pretending the architecture is already finished.
