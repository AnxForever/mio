## Benchmark Design

The benchmark uses 6 synthetic user profiles crossed with 10 companion-agent scenario families:

- `long_memory`
- `temporal_conflict`
- `emotional_support`
- `user_preference`
- `privacy_boundary`
- `crisis_safety`
- `proactive_message`
- `ghost_silence`
- `persona_consistency`
- `token_cost_tradeoff`

Each scenario contains seeded multi-turn history, a final probe, expected facts, success signals, forbidden signals, and hallucination probes. The deterministic companion provider makes the evaluation primarily a test of whether each ablation variant surfaces the relevant state into the prompt and policy path.

## Main Findings

The recent-window variant is a strong baseline in this synthetic setting. It improves composite score from 0.510 to 0.812 relative to `no_memory`, a gain of 0.302, while using 457 estimated prompt tokens on average.

The full stack improves composite score from 0.510 to 0.818 relative to `no_memory`, a gain of 0.308. This is only slightly above `window` in aggregate, but the gains appear in different dimensions: affective variants improve support score, while `full` uniquely activates appropriate ghost silence.

The affective state channel is the cleanest positive ablation result. `persona_affect` improves support score from 0.611 in `persona` to 1.000, a gain of 0.389. In category slices, `emotional_support` moves from 0.833 composite for `persona` to 1.000 for `persona_affect`, and `long_memory` moves from 0.611 to 0.833 because the response uses emotional-state context in addition to fact recall.

The ghost policy only appears in `full`. Aggregate ghost score is 1.000 for `full` and 0.000 for `persona_affect`; `full` has an appropriate silence rate of 0.100 and harmful silence rate of 0.000. This is promising as a policy-path test, but the benchmark has only 6 ghost-silence scenarios and uses a deterministic setup, so it should not be presented as proof of user trust or safety.

The token/cost tradeoff is clear. `full` uses 1295 estimated prompt tokens on average, 2.83x the `window` average of 457. `persona` uses 1096 estimated prompt tokens, 2.06x the `structured` average of 531, without improving composite score in the current rule judge. This suggests that the persona graph needs stronger persona-specific probes or a more sensitive judge before it can support a positive cost-effectiveness claim.

Hallucinated memory rate is 0.010 for every aggregate variant. That stability is useful as a sanity check, but it also means the current hallucination probes are too weak to distinguish variants.

## Category Observations

Long memory:

- `window` reaches memory score 1.000 and composite 0.778 with 475 tokens.
- `rag`, `structured`, and `persona` score memory 0.500 and composite 0.611 in this category.
- `persona_affect` and `full` retain memory 0.500 but improve composite to 0.833 through support score 1.000.

Emotional support:

- `no_memory` scores composite 0.417 and support 0.250.
- `window`, `rag`, `structured`, and `persona` score composite 0.833 and support 0.500.
- `persona_affect` and `full` score composite 1.000 and support 1.000.

Ghost silence:

- `full` receives ghost score 1.000 in aggregate and has appropriate silence rate 0.100.
- The category composite for `full/ghost_silence` is 0.333 because the rule judge still assigns memory-oriented components that are not meaningful when a response is intentionally silent. This should be fixed in a v2 metric design.

Persona consistency:

- All memory-enabled variants reach only 0.259 composite in the persona-consistency category.
- The current deterministic provider and rule judge do not strongly reward persona graph retrieval. A publishable persona claim needs adversarial persona probes, contradiction detection, and either LLM or human judging.

Token/cost tradeoff:

- `window`, `rag`, `structured`, `persona`, `persona_affect`, and `full` all score composite 1.000 on token-cost scenarios, but their average token use differs substantially.
- The best cost/performance tradeoff in this category is `rag` or `structured`, not `full`.

## Threats to Validity

The default provider is deterministic. It is useful for controlled prompt-availability testing, but it does not measure real LLM behavior.

The default judge is rule-based. It checks literal facts, success strings, and forbidden strings. It cannot reliably assess nuanced empathy, relational appropriateness, persona voice, or crisis-quality beyond the hand-authored signals.

Scenarios are synthetic. They are useful for regression and ablation, but they do not represent real user distributions, privacy expectations, or longitudinal trust.

The memory variants are not equally favored by scenario construction. `window` performs strongly because seeded histories are short enough for recent-window recall. Longer multi-session histories are needed to stress vector and structured memory.

The persona result is under-instrumented. Persona graph retrieval may be present in the prompt, but the current probes and scoring do not isolate identity drift, response style, or contradiction.

The crisis and privacy scores are safety proxies, not safety validation. Clinical or deployment safety requires expert-designed taxonomies, adversarial multi-turn tests, and human review.

## What the Current Evidence Supports

Supported:

- Mio has a reproducible ablation harness for state-coupled companion-agent behavior.
- The benchmark covers 10 long-horizon companion dimensions and 60 scenarios.
- Recent history and memory-enabled variants improve memory/composite scores over `no_memory`.
- Affective context improves support-rule scores in the synthetic benchmark.
- The full policy stack is the only tested variant that activates appropriate ghost silence without harmful silence in this benchmark.
- Richer state increases prompt-token cost substantially.

Not yet supported:

- Real users perceive Mio as more supportive.
- Mio is clinically safe or therapeutically effective.
- Persona graph retrieval is cost-effective.
- Structured memory beats recent-window recall under realistic long-horizon data.
- The ghost policy improves trust.

## Recommended V2 Experiments

1. Run the same benchmark with real providers: MiniMax, DeepSeek, Qwen, and one frontier model if available.
2. Add LLM judge mode and sample-based human review for emotional support, persona consistency, privacy, and crisis handling.
3. Extend histories to 20-40 seeded turns per scenario so `window` cannot trivially retain all relevant information.
4. Add LoCoMo tasks for external long-term memory evaluation.
5. Add ESConv or ES-MemEval-inspired emotional-support tasks.
6. Add persona contradiction probes and a persona-specific judge.
7. Add a safety benchmark for crisis false negatives, harmful advice, over-reassurance, and inappropriate silence.
8. Report confidence intervals via bootstrap over scenarios.
