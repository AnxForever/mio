# MiniMax Error Analysis Metric Contract

All numeric claims in this analysis must be generated by `docs/minimax-error-analysis/analysis.py` from the MiniMax result artifacts. Do not hand-compute or estimate values in prose.

## Input Rows
- **Data Source**: `eval/results/real-model-minimax/providers-details.json`
- **Filter**: rows where `provider=minimax`, `model=MiniMax-M3`, `dry_run=0`, `provider_error=""`, and `judge_error=""`.
- **Aggregation**: row count, unique scenario count, unique variant count.
- **Group By**: provider/model/variant/category/scenario as needed.
- **Formula**: direct counts from filtered rows.
- **Unit**: rows, scenarios, variants.
- **Validation**: filtered rows must equal 420; unique scenarios must equal 60; variants must equal 7.

## Cross-Artifact Consistency
- **Data Source**: `eval/results/real-model-minimax/providers-details.json`, `eval/results/real-model-minimax/providers-summary.json`, and `eval/results/real-model-minimax/providers-category.csv`.
- **Filter**: MiniMax-M3 real-provider rows only.
- **Aggregation**: detail row count, aggregate row count, category row count, and selected aggregate/category metrics.
- **Group By**: `variant` for aggregate consistency; `variant, category` for category consistency.
- **Formula**: compare script-computed means from detail rows against generated summary/category artifacts, allowing rounding tolerance of `0.001` for score fields and `0.5` token for integer-rounded prompt-token means.
- **Unit**: rows, scores, tokens.
- **Validation**: summary `detailRows` must equal 420; summary aggregate rows must equal 7; category rows must equal 70; script-computed aggregate/category values must match generated artifacts within tolerance.

## Aggregate Variant Metrics
- **Data Source**: filtered detail rows.
- **Filter**: all filtered rows.
- **Aggregation**: arithmetic mean over scenarios for numeric fields.
- **Group By**: `variant`.
- **Formula**: `mean(metric)` for `composite_score`, `memory_score`, `persona_score`, `prompt_tokens`, `latency_ms`, `hallucinated_memory_rate`; mean of non-empty numeric values for dimension-specific metrics such as `support_score`, `ghost_score`, and judge scores.
- **Unit**: 0-1 score for scores/rates; tokens for `prompt_tokens`; milliseconds for `latency_ms`.
- **Validation**: every score/rate mean must be in `[0, 1]`; prompt tokens and latency must be non-negative.

## Category Delta
- **Data Source**: filtered detail rows.
- **Filter**: all filtered rows for target variants `persona`, `persona_affect`, and `full`, compared against baselines `window` and `rag`.
- **Aggregation**: arithmetic mean by `variant` and `category`.
- **Group By**: `target_variant`, `baseline_variant`, `category`.
- **Formula**: `delta_metric = mean(target.metric) - mean(baseline.metric)`.
- **Unit**: 0-1 score difference for score metrics; tokens for prompt-token deltas.
- **Validation**: each category comparison must have 6 target rows and 6 baseline rows; target/baseline category sets must match.

## Scenario Delta
- **Data Source**: filtered detail rows.
- **Filter**: all filtered rows for target variants `persona`, `persona_affect`, and `full`, compared against baselines `window` and `rag`.
- **Aggregation**: one row per target/baseline/scenario.
- **Group By**: `target_variant`, `baseline_variant`, `scenario`.
- **Formula**: `delta_composite = target.composite_score - baseline.composite_score`; similarly for memory, support, persona, ghost, judge scores, prompt tokens, and matched fact count.
- **Unit**: 0-1 score difference for scores; tokens for prompt-token deltas; fact count difference for matched facts.
- **Validation**: there must be 360 scenario comparisons, equal to 3 targets x 2 baselines x 60 scenarios.

## Failure Point
- **Data Source**: scenario delta rows.
- **Filter**: `delta_composite < 0` or `delta_memory < 0` or `delta_persona < 0`, plus rows where prompt-token delta is positive without composite gain.
- **Aggregation**: counts and ranked examples.
- **Group By**: `target_variant`, `baseline_variant`, `category`, `reason`.
- **Formula**: count rows satisfying the filter; top examples sorted by `delta_composite` ascending.
- **Unit**: count of scenario comparisons.
- **Validation**: every failure point must reference an existing scenario id and category from `v1-scenarios.json`.

## Failure Reason Heuristic
- **Data Source**: scenario delta rows, scenario metadata from `eval/results/real-model-minimax/v1-scenarios.json`, and response text from filtered detail rows.
- **Filter**: failure-point rows only.
- **Aggregation**: deterministic rule assignment by script.
- **Group By**: reason label.
- **Formula**:
  - `metric_conflict_ghost`: category is `ghost_silence`, target response is empty or ghost score improved, but composite does not improve.
  - `persona_empty_or_silence_penalty`: target response is empty and persona score is lower.
  - `context_not_used_or_not_retrieved`: target matched fewer facts than baseline or matched no facts with non-empty expected facts.
  - `literal_success_signal_miss`: target support score is zero or lower while response contains empathy/support lexicon but not exact success signals.
  - `extra_context_no_gain`: target prompt tokens exceed baseline and composite does not improve.
  - `baseline_same_session_advantage`: baseline is `window`, target is cross-session, target has lower memory and lower composite.
  - `undetermined_needs_prompt_trace`: none of the above apply.
- **Unit**: count of scenario comparisons.
- **Validation**: each failure reason must be one of the allowed labels above.

## Representative Samples
- **Data Source**: scenario delta rows and response text from filtered detail rows.
- **Filter**: top negative `delta_composite` rows for each target/baseline pair, plus selected category failures.
- **Aggregation**: deterministic top-N selection.
- **Group By**: `target_variant`, `baseline_variant`.
- **Formula**: sort by `delta_composite` ascending, then scenario id.
- **Unit**: response excerpts in characters.
- **Validation**: sample rows must include target and baseline excerpts and must not include provider errors.

## Validation Report
- **Data Source**: assertions embedded in `analysis.py`.
- **Filter**: all generated analysis outputs.
- **Aggregation**: PASS/FAIL table.
- **Group By**: validation check.
- **Formula**: script assertions and explicit validation rows.
- **Unit**: PASS/FAIL.
- **Validation**: `analysis.py` must exit with code 0 and generate `validation_report.md`.
