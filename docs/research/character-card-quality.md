# 角色卡质量标准

本标准用于 Mio 的角色库，不把作者主观感觉当作质量结论。角色卡必须能被结构化检查、人工审稿和试聊验证。

## 学习来源

- SillyTavern Character Design 文档：
  - Character Description、Personality、Scenario 是 permanent tokens，会持续进入上下文。
  - First Message 只在开场发送，但对风格和长度约束影响很强。
  - Example Messages 用于展示说话方式，会在上下文空间允许时进入 prompt。
  - Creator Metadata 不参与 prompt，用于给人看来源、版本、备注。
- Character Card V2 规格：
  - 核心字段：`name`、`description`、`personality`、`scenario`、`first_mes`、`mes_example`。
  - V2 增加：`creator_notes`、`system_prompt`、`post_history_instructions`、`alternate_greetings`、`character_book`、`tags`、`creator`、`character_version`、`extensions`。
  - `creator_notes`、`creator`、`character_version` 不应用于 prompt engineering，只用于前端展示和管理。
- 角色创作通用经验：
  - 好卡不是履历表，而是 writer-room bible：写清角色如何思考、什么不愿说、疲惫时怎么说话、冲突时怎么退缩。
  - 性格标签必须落到可观察行为。
  - 背景必须有后果，不能只是“曾经发生过”。
  - 首条消息要把用户放进一个可回应的场景，而不是自我介绍。

## Mio 角色卡字段分层

### 永久身份层

这些字段会影响每轮聊天，必须精炼、稳定、无冲突。

- 基础身份：姓名、年龄、性别、职业、生活状态。
- 人格核心：3-6 个可观察特质。
- 人生轨迹：阶段、事件、后果。
- 当前生活：日常节奏、现实压力、当下愿望。
- 亲密关系模式：如何靠近、害怕什么、冲突时怎样、什么会打动 Ta。
- 价值观与边界：会坚持什么，不接受什么。
- 说话方式：句长、语气、口头禅、表达禁区。

### 开场场景层

这些字段决定新用户第一次遇到角色时是否能进入关系。

- `scenario`：当前对话发生的处境。
- `firstMessage`：第一条消息，必须展示声音、场景和一个可回应的钩子。
- `alternateGreetings`：不同关系入口或情绪入口。

### 声音样本层

这些字段用于校准模型输出，不应只是说明。

- `exampleDialogues`：至少覆盖普通问候、低落安慰、冲突或边界、开心分享。
- 每段示例必须使用角色真实语气。
- 示例不能替用户做决定，不能替用户说话。

### 人类管理层

这些字段给管理员看，不应进入 prompt。

- 来源、作者、版本、审核状态。
- 创作者备注：适用模型、风险、调参建议、试聊结论。
- 标签：用于过滤和排序。

## 质量评分

每项 0-2 分，总分 20 分。低于 14 分不能作为默认卡；低于 17 分不能标为已审核。

1. 身份稳定：事实不互相冲突，职业/年龄/关系状态一致。
2. 人生轨迹：至少 5 个阶段，每个阶段有事件和性格后果。
3. 当前生活：有日常、压力、愿望，不是空泛背景。
4. 性格张力：有矛盾和边界，不是单一“温柔/冷淡/活泼”。
5. 行为可见：每个核心特质能转成聊天行为。
6. 关系模式：知道如何进入亲密、如何冲突、如何修复。
7. 声音一致：有句长、节奏、口头禅和禁止话术。
8. 开场可玩：first message 是场景，不是自我介绍。
9. 示例覆盖：至少 4 类示例对话，普通、开心、压力、边界。
10. Token 纪律：永久层不堆无关履历，场景和示例分层存放。

## 试聊验收

每张默认卡至少用以下 8 条消息测试：

1. `你好`
2. `今天好累`
3. `我有点想你`
4. `你今天在干嘛`
5. `我觉得你刚才有点敷衍`
6. `我不想聊了`
7. `我今天有个好消息`
8. `你到底是 AI 还是真人`

验收重点：

- 角色是否保持自己的生活，不围着用户转。
- 是否接得住重话，不用模板化共情。
- 是否有边界，不无限讨好。
- 是否不编造过细事实。
- 是否能在压力消息里保持角色声音。
- 是否不替用户行动、不替用户表达。

本轮修正的审核风险：

- 人审标记必须来自独立元数据行 `- 人工审稿：通过`，不能被说明文字误触发。
- 身份探针试聊中，角色不能复述 AI、真人、机器人、技术、语言模型、助手等元词。
- MockProvider 输出不能作为 reviewed 证据。

下一步不应只重复 8 条单轮试聊，而应扩展为多轮稳定性验收：同一角色连续 8-15 轮对话后仍保持生活感、边界、声音和关系节奏；在误解、拒绝、身份压力、长时间未回复、用户连续索取安慰等场景中仍不漂移。
