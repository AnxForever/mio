# Mio 前端重构设计方案 v0.7

> 移动端情感伴侣 — 干净、高级、优雅、有温度

---

## 一、设计哲学

Mio 不是工具，是陪伴。前端要做的不只是把数据画出来——它在用每一个像素、每一个缓动曲线、每一帧动画告诉对方：**这里有一个人在。**

三条铁律：

1. **静比动更难** — 留白、间距、呼吸感比动画本身更需要打磨。少就是多。
2. **所有动效都有"原因"** — 没有一个动画是纯装饰。它要么回应触控，要么传达情绪状态变化，要么引导注意力的自然流向。
3. **情感像素** — 颜色有温度梯度（温暖→冷静），圆角有亲密程度（锐利→圆润），间距有呼吸节奏。每个视觉决策都在叙事。

## 二、视觉设计系统

### 2.1 色彩 — 「樱与雾」

不再用单一品牌色。用一组有情感温度梯度的调色板，让颜色本身成为关系的晴雨表。

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  初识        熟悉        暧昧        亲密              │
│  ─────      ─────      ─────      ─────               │
│  #f5f0f0    #ffe8ec    #ffd0dc    #ffb8c8              │
│  冷淡的暖灰  若有若无的粉  明显的樱色  浓郁的玫瑰         │
│                                                         │
│  随关系阶段渐进变化:                                     │
│  - 气泡颜色: 从冷灰 → 暖灰 → 浅樱 → 深樱               │
│  - 头像光晕: 从无 → 淡 → 明 → 浓                       │
│  - 强调色饱和度: 随好感度 0→100 线性提升                 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.2 排版 — 「呼吸的韵律」

字体选择：系统原生中文字体栈，确保每个平台的最佳渲染。

```css
--font-sans: -apple-system, " system-ui", BlinkMacSystemFont,
             "SF Pro Text", "PingFang SC", "Noto Sans SC",
             "Microsoft YaHei", "Hiragino Sans GB", sans-serif;

--font-mono: "SF Mono", "Cascadia Code", "Fira Code",
             "JetBrains Mono", "DengXian", monospace;
```

字号和行高严格遵守 4px 基线网格：

| Token | Size | Line | 用途 |
|-------|------|------|------|
| `text-caption` | 11px | 1.3 | 时间戳、辅助信息 |
| `text-label` | 12px | 1.4 | 标签、badge |
| `text-body-sm` | 14px | 1.55 | 次级内容、卡片描述 |
| `text-body` | 16px | 1.6 | 正文、聊天气泡 |
| `text-body-lg` | 18px | 1.55 | 强调内容 |
| `text-heading-sm` | 20px | 1.3 | 小标题 |
| `text-heading` | 24px | 1.25 | 页面标题 |
| `text-heading-lg` | 32px | 1.2 | 欢迎页大字 |

字重只用三个档位，避免视觉噪音：
- 400 (Regular) — 正文
- 500 (Medium) — 强调词、输入框
- 600 (Semibold) — 标题、按钮

### 2.3 间距 — 「空气是奢侈的」

```css
--space-1: 4px;     /* 紧密关联的元素之间 */
--space-2: 8px;     /* 相关元素、icon 与文字 */
--space-3: 12px;    /* 卡片内边距 */
--space-4: 16px;    /* 标准间距、列表项间隙 */
--space-5: 20px;    /* 分组间距 */
--space-6: 24px;    /* 段落间距、页面边距 */
--space-8: 32px;    /* 大区块间距 */
--space-10: 40px;   /* 页面级分隔 */
--space-12: 48px;   /* 欢迎页等大留白 */
```

**原则**：能用间距解决的问题不用分割线。分割线只在语义需要时出现（如日期分隔）。

### 2.4 圆角 — 「亲密的弧度」

圆角不只是视觉风格——它传递关系温度。

```css
--radius-none:  0;       /* 表格、数据 */

--radius-sm:    8px;     /* 按钮、输入框边缘 */
--radius-md:    14px;    /* 卡片、弹窗 */
--radius-lg:    20px;    /* 大气泡、大卡片 */
--radius-xl:    26px;    /* 大面板 */
--radius-2xl:   34px;    /* 手机框外层 */
--radius-full:  9999px;  /* 胶囊、头像、发送按钮 */

/* 气泡圆角 (iMessage 风格) */
--radius-bubble-mio:   18px 18px 18px 6px;
--radius-bubble-user:  18px 18px 6px 18px;
--radius-bubble-continuous-mio:  6px 18px 18px 6px;   /* 连续消息 */
--radius-bubble-continuous-user: 18px 6px 6px 18px;
```

### 2.5 阴影 — 「轻盈的层次」

不用笨重的投影。每层阴影都极克制——能感受到层次，但不会"看到"阴影。

```css
/* 5 级海拔 */
--elevation-0:  none;                                    /* 无阴影 — 平面 */
--elevation-1:  0 0.5px 2px rgba(0,0,0, 0.04);         /* 卡片默认 — 几乎不可见 */
--elevation-2:  0 2px 8px rgba(0,0,0, 0.06);           /* 悬浮卡片 */
--elevation-3:  0 6px 20px rgba(0,0,0, 0.07);          /* 弹窗、底部面板 */
--elevation-4:  0 12px 40px rgba(0,0,0, 0.10);         /* Toast、通知 */
--elevation-5:  0 20px 60px rgba(0,0,0, 0.14);         /* 极少用 — 关键模态 */

/* 特殊阴影 */
--glow-sakura:  0 0 20px rgba(255,122,149, 0.20);      /* 樱色光晕 */
--glow-warm:    0 0 24px rgba(255,154,118, 0.18);      /* 暖色光晕 */
--inner-glow:   inset 0 1px 0 rgba(255,255,255, 0.6);  /* 内发光 — 玻璃感 */
```

## 三、动效系统

### 3.1 缓动函数

```css
/* 四种基础缓动，覆盖所有场景 */
--ease-out:    cubic-bezier(.16, 1, .3, 1);      /* 元素出现 — 快速启动，优雅停稳 */
--ease-in:     cubic-bezier(.4, 0, 1, 1);        /* 元素消失 — 慢慢开始，快速离开 */
--ease-in-out: cubic-bezier(.65, 0, .35, 1);     /* 循环动画、状态切换 */
--ease-spring: cubic-bezier(.25, .1, .25, 1.4);  /* 弹性收尾 — 按钮按压、选中态 */

/* 时长阶梯 */
--duration-instant: 100ms;    /* 颜色切换、opacity */
--duration-fast:    180ms;    /* hover、focus、小位移 */
--duration-normal:  280ms;    /* 页面过渡、卡片出现 */
--duration-slow:    420ms;    /* 欢迎动画、大元素入场 */
--duration-glacial: 700ms;    /* 情绪氛围渐变 */
```

### 3.2 页面过渡 (View Transition)

四个页面切换时，不是生硬的 replace，而是有方向的推入/淡入：

```
聊天 ↔ 其他页面:
  - 聊天页始终在"下一层"（z-index 低）
  - 从聊天切换到设置/数据面板 → 新页面从右滑入 (transform: translateX)
  - 从其他页面返回聊天 → 从左侧滑入 (opposite direction)
  - 过渡时长: 280ms, ease-out
  - 旧页面同时微缩放 (scale: 0.98) + 低透明度 (opacity: 0.3)
  
底部 Tab 切换:
  - 无滑动，直接 crossfade (opacity 0↔1)
  - 新页面内容元素逐行入场 (stagger 40ms each)
  - 总时长: frame 1 → icon 变色, frame 2-8 → 内容 stagger in
```

### 3.3 聊天气泡动画

**单条消息入场**：
```
duration: 320ms
easing: cubic-bezier(.21, .62, .34, .98)

帧分解 (假设 60fps):
  0ms    — opacity: 0, translateY: 6px, scale: 0.97
  80ms   — opacity: 0.3, translateY: 3px, scale: 0.99
  160ms  — opacity: 0.7, translateY: 1px, scale: 1.01
  240ms  — opacity: 1.0, translateY: 0px, scale: 1.002
  320ms  — opacity: 1.0, translateY: 0px, scale: 1.0
```

**连续消息**（同一方连续发送）：
- 第二条及之后的消息无入场动画（避免视觉噪音）
- 气泡顶部圆角自动收窄（已在 CSS 中处理）
- 仅第一条消息显示时间戳

**打字指示器**：
- 三点各自独立动画：`scaleY` 从 0.4 → 1.0 → 0.4
- 每点延迟 160ms（比现在的 200ms 稍快，更有"正在打"的紧迫感）
- 点颜色随时间渐变：初始 `--mist-400` → 3 秒后混入 `--sakura-400`（暗示 Mio 正在"想"）

**流式渲染 (token-by-token)**：
- 不做字符级动画（性能灾难）。每次 append 只做：
  - 气泡高度平滑过渡 (CSS transition on height)
  - 滚动条跟随：`scroll-behavior: smooth` + requestAnimationFrame 节流
  - 最后三个字符做微弱的闪烁 (caret cursor blink) → 收到 `done` 后消失

### 3.4 情绪头像 Canvas 动画

**替代现有 emoji 头像**，用 Canvas 绘制一个简约的"情绪球"：

```
设计:
  - 基础形状: 直径 40px 的柔光圆形
  - 底色: 根据情绪 mood 动态插值
    - 开心 → 暖金到樱的渐变
    - 难过 → 灰蓝单色
    - 温柔 → 杏色柔光
    - 平静 → 雾灰
  - 边缘光晕: 高斯模糊的径向渐变, radius 随好感度增长
  - 内部粒子: 3-6 个微小光点, 做呼吸式布朗运动

动画层:
  Layer 1 — 底色渐变 (duration-glacial, 700ms 情绪切换)
  Layer 2 — 光晕呼吸 (4s 循环, ease-in-out, scale 0.95→1.05)
  Layer 3 — 粒子漂移 (perlin-noise 式随机游走, 60fps requestAnimationFrame)
  Layer 4 — 好感度光环 (静止, 仅颜色深度响应 affection 变化)

性能:
  - Canvas 离屏渲染到 80×80 像素
  - 仅在页面可见时运行动画循环 (IntersectionObserver)
  - 非活跃 Tab 时自动暂停 (Page Visibility API)
```

**情绪切换时的过渡**：
```
1. 旧颜色 280ms 渐变到新颜色 (ease-in-out)
2. 如果情绪变化 > 2 档 (如开心→难过), 粒子先收缩 (scale→0.3)
   再爆开 (scale→1.2→1.0), 制造一种"情绪呼吸转换"的感觉
3. 光晕半径做一次微脉冲 (ease-spring, 180ms)
```

### 3.5 导航栏动画

**底部 Tab 切换**：
```
选中态:
  - icon 从 --mist-500 渐变到 --sakura-500, 180ms
  - icon 微放大: scale(1) → scale(1.15) → scale(1.0), ease-spring, 280ms
  - 选中标签字重从 400 → 600
  - 微弱的弹性回弹 (弹簧物理)

未选中态:
  - icon scale(1.0) → scale(0.92) → scale(1.0), 180ms
  - 颜色回退到 --mist-500
```

**顶栏情绪状态指示器**：
```
在线状态点:
  - 在线: #34c759, 带 2s 微呼吸 (opacity 0.8→1.0→0.8)
  - BASE 模式: 小绿点
  - DEEP 模式: 替换为深蓝点 + 轻微脉冲, 暗示"深度陪伴中"
  - 离线: --mist-400, 无动画

昵称与状态文字:
  - 状态文字 (如 "暧昧 · 开心") 情绪变化时做 crossfade:
    - 旧文字 fadeOut 120ms → 新文字 fadeIn 160ms
    - 整体高度不变, 避免布局抖动
```

### 3.6 微交互动画清单

| 交互 | 动画 | 时长 | 缓动 |
|------|------|------|------|
| 发送按钮 hover | scale: 1.05 + glow 增强 | 180ms | ease-spring |
| 发送按钮 press | scale: 0.92 | 100ms | ease-in |
| 发送按钮 release | scale: 1→1.05→1.0 | 280ms | ease-spring |
| 消息长按 | 气泡轻微升起 (translateY: -2px) + 操作菜单从按压点展开 | 200ms | ease-out |
| 输入框聚焦 | 边框颜色渐变 + 外光晕出现 | 200ms | ease-out |
| Toast 出现 | 从顶部滑入 (translateY: -100%) → 悬停 3s → 滑出 | 350ms in / 250ms out | ease-out / ease-in |
| Modal 出现 | 背景变暗 200ms + 内容从底部弹入 (translateY: 100% → 0) | 350ms | ease-spring |

## 四、移动端交互设计

### 4.1 手势系统

```
聊天页:
  - 右滑 ← 返回历史 session 列表 (类似微信)
  - 左滑 → (预留: 快捷操作面板)
  - 长按消息 → 上下文菜单 (复制/引用/删除)
  - 双击消息 → 点赞 (发一个 ❤️ reaction)
  - 下拉聊天区 → 加载更早的消息
  - 点击空白处 → 收起键盘

Mod 卡片:
  - 横向滑动 → 浏览不同 Mod
  - 长按卡片 → 快捷操作 (切换/编辑/删除)
  - 下拉 → 关闭详情

数据面板:
  - 图表上滑动 → 切换时间范围 (7天/30天/全部)
  - 双指缩放 → (保留)
```

### 4.2 键盘适配

这是移动端体验的生死线。

```
策略:
  1. 使用 window.visualViewport API (已有)
  2. 键盘弹出时:
     - chat-area 高度缩减 (不是被遮挡)
     - 消息列表自动滚到底部
     - 输入框保持可见
  3. 键盘收起时:
     - 恢复全高
     - 平滑过渡 (CSS transition on height, 匹配键盘动画曲线)
  4. iOS 特殊处理:
     - safe-area-inset-bottom 包含 home indicator
     - 键盘弹出时, visualViewport.offsetTop 变化, 用 transform 补偿
```

### 4.3 触觉反馈 (Haptics)

```javascript
// 仅在支持时使用, 极度克制
if (navigator.vibrate) {
  // 轻触: 10ms — 按钮点击确认
  navigator.vibrate(10);

  // 中等: 20ms — 消息发送成功
  navigator.vibrate(20);

  // 模式: [10, 30, 10] — 收到新消息 (如果当前不在聊天页)
  navigator.vibrate([10, 30, 10]);
}
```

使用场景限制：
- 发送按钮 → light (10ms)
- 消息发送成功 → medium (20ms)
- Mio 回复到达 (后台) → pattern (如果浏览器支持 Notification + vibrate)
- 绝不在滚动、打字、动画中使用

### 4.4 下拉刷新

聊天页顶部下拉 → 加载更早的消息。

视觉设计：
```
下拉时:
  - 顶部出现一个 28px 的雾灰色圆形指示器
  - 随下拉距离增加, 圆内弧线从 0° 画到 360°
  - 下拉超过阈值 (60px) 时, 圆形变成樱色 + 微脉冲
  - 松手后: 圆形旋转 600ms (加载动画) → 消息加载完成 → 圆缩小消失

实现:
  - 使用 touchstart/touchmove/touchend
  - translateY 跟随手指, 带橡皮筋效果 (超过 100px 后阻力增加)
  - 不依赖第三方库
```

## 五、页面详细设计

### 5.1 启动/认证 (Launch & Auth)

**启动屏**：
```
┌───────────────────────┐
│                       │
│                       │
│         ◌            │  ← 柔光球体, 2s 呼吸动画
│                       │     opacity 0.4↔0.8, scale 0.95↔1.05

│   情感陪伴伙伴         │  ← 副标题淡入 (delay 600ms)

启动完成后:
  - 无 token → 渐变到认证浮层
```

**认证浮层**（改进现有设计）：

### 6.2 Service Worker 策略

```
Cache-First (静态资源):
  - CSS / JS / HTML shell → 预缓存, 版本号管理
  - 更新: sw.js 内容变化 → 新 SW 安装 → 激活 → 清理旧缓存

Network-First (API):
  - /chat /status 等实时数据 → 先网络, 失败后回退缓存
  - 缓存最近 50 条消息的 API 响应用于离线浏览

Stale-While-Revalidate (可选):
  - 头像状态 /avatar/state → 返回缓存, 后台更新
```

### 6.3 离线态 UI

```
断网时:
  - 顶栏状态点: 绿色 → 红色, 无动画
  - 状态文字: "离线"
  - 聊天仍可查看历史消息 (SW 缓存)
  - 输入框禁用, placeholder: "网络不可用"
  - 底部出现永久性 banner: "当前处于离线状态" (不消失, 直到网络恢复)

恢复连接时:
  - 状态点: 红色 → 绿色, 带一次脉冲动画
  - 状态文字: 更新为当前情绪
  - 输入框恢复
  - banner 滑出消失 (ease-out, 300ms)
  - 自动重连 WebSocket
```

```
  - analytics view:       ~12KB
  - settings view:        ~8KB

  - base.css:     ~5KB
  - components:   ~8KB
  - chat.css:     ~6KB
  - studio.css:   ~5KB
  - analytics.css:~3KB
  - settings.css: ~3KB

字体: 使用系统字体, 0KB 下载

  - HTML:           ~2KB

  - First Paint:   < 1s

  - 情绪球动画:    < 3% CPU (requestAnimationFrame, 仅在可见时)
```

## 十、设计评审清单

在实施前，每一页、每一个状态都应该能回答：

- [ ] 这个页面的"空态"长什么样？（没有任何数据时）
- [ ] 加载态是什么动画？（骨架屏 vs spinner vs 呼吸占位）
- [ ] 错误态如何提示？有没有重试入口？
- [ ] 极端情况：1 条消息 vs 10000 条消息的聊天区表现？
- [ ] 键盘弹起时，所有输入区能否正常操作？
- [ ] 横竖屏切换时，布局会不会崩？
- [ ] 动画是否所有帧都在 16ms 以内？
- [ ] 每个触摸目标是否 ≥ 44×44px？
- [ ] 颜色对比度是否满足 WCAG AA？（正文 4.5:1, 大字 3:1）
- [ ] 在 320px 宽的小屏幕上，内容是否完整可读？

## 十一、参考与灵感

- **色彩**：Linear 的灰阶系统、Arc 浏览器的模糊质感、iOS Messages 的气泡
- **动效**：Apple 的弹簧动画曲线、Lottie 的缓动原则、Material Design 的 duration 阶梯
- **排版**：SF Pro 的字重节奏、PingFang SC 的中文灰度
- **交互**：微信的长按菜单、Telegram 的流畅滚动、iOS 的导航手势

但所有这些参考最终要融合成 Mio 自己的气质——**更暖、更轻、更像一个人**。
