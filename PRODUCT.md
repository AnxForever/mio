# Product

## Register

product

## Users

Mio is used by local agent operators, developers, and technically comfortable creators who want to run and tune an emotional companion agent from their own machine. They open the interface to check runtime health, continue a conversation, edit persona material, review memory, inspect analytics, and understand which extensions or integrations are available.

## Product Purpose

Mio provides a focused control surface for an emotional companion agent: chat, persona configuration, relationship state, memory governance, notification setup, and extension visibility in one local web UI. Success means the operator can understand the agent's current state quickly, make careful changes without guessing, and return to conversation without losing context.

## Brand Personality

Precise, intimate, composed.

The product should feel like a professional AI console with a quiet emotional core. It should be refined and tactile, but not ornamental. It can reveal Mio's companion identity through small stateful details, not through decorative clutter.

## Anti-references

Do not make it look like a marketing landing page, a toy chatbot skin, a stock SaaS dashboard, a neon cyberpunk console, or an over-animated demo. Avoid decorative gradients, large hero sections, loud color themes, fake plugin controls, and UI that suggests backend capabilities that are not implemented.

## Design Principles

1. Keep the operator oriented: every page should make location, state, and next action obvious.
2. Make configuration feel deliberate: persona, memory, skills, plugins, and MCP surfaces need clear states and careful wording.
3. Let Mio be present without turning the console into a character page: emotional details should appear as status, mood, avatar, and relationship signals.
4. Use restraint as craft: hierarchy, spacing, motion, and text should do the work before color or decoration.
5. Never fake capability: pending integrations must read as planned or unavailable, not as clickable working controls.

## Accessibility & Inclusion

Target WCAG AA for contrast and keyboard access. Respect reduced-motion preferences. Keep touch targets stable on mobile, avoid text overflow in controls, and use visible focus states for all interactive elements.
